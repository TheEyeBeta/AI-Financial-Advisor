# Repository Structure

This document describes the organized structure of the AI Financial Advisor repository.

## Directory Organization

```
AI-Financial-Advisor/
├── .github/
│   └── workflows/          # CI/CD pipelines
├── backend/
│   └── websearch_service/  # Python FastAPI backend
│       ├── app/            # Application code
│       ├── tests/          # Backend tests
│       ├── Dockerfile      # Backend Docker image
│       └── requirements.txt
├── config/                 # Configuration templates
│   ├── env.example         # Local development env template
│   └── env.production.example  # Production env template
├── deployment/             # Deployment configurations & documentation
│   ├── DEPLOYMENT.md       # Comprehensive deployment guide
│   ├── DEPLOYMENT_CHECKLIST.md  # Pre-deployment checklist
│   ├── docker-compose.yml  # Local Docker Compose setup
│   ├── Dockerfile.frontend # Frontend Docker image
│   ├── nginx.conf          # Nginx configuration
│   └── deploy.sh           # Deployment helper script
├── docs/                   # Documentation
│   ├── tests/              # Test documentation
│   │   └── TEST_SUMMARY.md
│   ├── security/           # Security documentation
│   ├── curriculum-implementation.md
│   └── learning-system-architecture.md
├── e2e/                    # End-to-end tests (Playwright)
├── public/                 # Static assets
├── scripts/                # Utility scripts
├── src/                    # Frontend React application
│   ├── components/        # React components
│   ├── pages/             # Page components
│   ├── lib/               # Utilities & helpers
│   ├── services/          # API services
│   └── tests/              # Frontend tests
├── sql/                    # Database schemas
├── railway.json            # Railway platform config (root for auto-detection)
├── render.yaml             # Render platform config (root for auto-detection)
├── vercel.json             # Vercel platform config (root for auto-detection)
├── package.json            # Frontend dependencies
├── Makefile                # Development commands
└── README.md               # Project overview
```

## File Organization Principles

### Root Directory
- **Platform configs** (`railway.json`, `render.yaml`, `vercel.json`) stay in root for auto-detection
- **Build configs** (`package.json`, `tsconfig.json`, `vite.config.ts`) stay in root for tooling
- **Main documentation** (`README.md`) stays in root

### `/deployment/` Folder
All deployment-related files:
- Docker configurations
- Deployment documentation
- Deployment scripts
- Platform-specific guides

### `/config/` Folder
Configuration templates and examples:
- Environment variable templates
- Configuration examples

### `/docs/` Folder
All documentation:
- Architecture docs
- Test documentation
- Security documentation
- Learning system docs

### `/scripts/` Folder
Utility scripts for development and maintenance

## Quick Reference

## Commands Reference

```bash
# Development
make dev                    # Start dev servers
make docker-up              # Start Docker services

# Testing
make test                   # Run all tests
npm test                    # Frontend tests
npm run test:e2e            # E2E tests

# Deployment
make deploy-frontend        # Deploy to Vercel
make deploy-backend         # Deploy to Railway
make deploy-all             # Deploy both

# Docker
docker-compose -f deployment/docker-compose.yml up -d
docker-compose -f deployment/docker-compose.yml logs -f
docker-compose -f deployment/docker-compose.yml down
```
